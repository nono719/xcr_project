# 架构说明

## 整体架构

```
┌──────────────────────────────┐         ┌──────────────────────────────┐
│       Browser (Chrome)       │ ──API──▶│      Express Backend         │
│  React + AntD + Monaco       │         │  /auth /courses /experiment  │
│  Vite dev proxy → :4000      │         │  /evaluation /teacher /admin │
└──────────────────────────────┘         └────────────┬─────────────────┘
                                                      │
                                  ┌───────────────────┼────────────────────┐
                                  │                   │                    │
                                  ▼                   ▼                    ▼
                          ┌──────────────┐   ┌────────────────┐   ┌─────────────────┐
                          │   MySQL 8    │   │ Ganache 子进程  │   │   solc 编译器    │
                          │  用户/课程   │   │ 每实验一实例    │   │  JIT 编译合约    │
                          │  案例/记录   │   │ 隔离端口/链ID   │   │                 │
                          └──────────────┘   └────────────────┘   └─────────────────┘
```

## 评测流水线

1. **接收** `POST /api/evaluation/submit { caseId, mode, code }`
2. **编译** 学生代码 + 漏洞合约（solc 0.8.20）
3. **启动沙箱** 拉起独立 Ganache 进程，分配预置账户
4. **部署** 漏洞合约 → 必要时调用 `deposit()` 注入余额
5. **部署** 学生合约（构造函数若是单 `address`，自动注入 victim 地址）
6. **执行**
   - attack 模式：调用 `attack()`，对比 victim 合约余额变化判定漏洞触发
   - fix 模式：部署学生合约（视为修复版），跑回归测试
7. **打分** 五维加权：编译 10 / 部署 10 / 执行 10 / 漏洞触发 35 / 防御有效 35
8. **持久化** 写入 `experiment_record`（detail JSON 保留分项明细）
9. **销毁沙箱** `SIGKILL` 子进程，释放端口

## 安全设计

- 密码 bcrypt 加盐哈希
- JWT 7 天有效期 + 请求头注入 + 401 自动登出
- 角色三级权限：student / teacher / admin
- 沙箱与主链严格隔离，所有操作仅在 Ganache 仿真链
- 登录失败 5 次锁定 15 分钟
- API 响应统一 `{ code, message, data }`
- 关键操作记录到 winston + 数据库 `op_log`

## 限流与并发

- Ganache 端口池：默认 8600–8799，可容纳 200 并发沙箱
- 沙箱 15 分钟空闲自动回收（GC 每分钟）
- 评测过程是临时沙箱：评测后立即销毁
- 长期实验沙箱通过 `/api/experiment/start` 保留，停止靠用户手动或 idle GC
