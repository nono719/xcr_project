# 智能合约安全威胁概览

## 威胁分类（SWC 体系）

SWC（**Smart Contract Weakness Classification**）是行业广泛使用的漏洞分类标准，类似传统 web 安全的 OWASP Top 10。本平台覆盖的 5 类对应：

| SWC | 漏洞 | 难度 | 危害 |
|---|---|---|---|
| **SWC-107** | 重入攻击 | ⭐ | 资金被掏空 |
| **SWC-101** | 整数算术问题 | ⭐⭐ | 任意发行 / 任意提现 |
| **SWC-114** | 抢先交易 | ⭐⭐ | 经济收益被劫持 (MEV) |
| **SWC-113** | 拒绝服务 (DoS) | ⭐⭐ | 合约功能瘫痪 |
| **SWC-115** | tx.origin 鉴权 | ⭐ | 越权操作 |

完整 SWC 列表有 130+ 项，但 80% 的真实损失都集中在上面几类。

## 三层威胁模型

```
┌─ Solidity 层 ──────────────────────────────┐
│  逻辑漏洞 / 类型错误 / 未检查返回值          │
│  ▲                                          │
├─ EVM 层 ──────────────────────────────────┤
│  操作码语义 / Gas 模型 / 委托调用          │
│  ▲                                          │
├─ 链层 ───────────────────────────────────┤
│  Mempool 公开性 / 矿工排序 / 时间戳        │
└─────────────────────────────────────────────┘
```

- **Solidity 层**：合约代码本身的逻辑缺陷——重入、整数溢出、未初始化变量
- **EVM 层**：操作码与 gas 机制特性——delegatecall 存储冲突、send/transfer 与重入的关系
- **链层**：交易顺序与外部环境——抢先交易、时间戳依赖

## 攻击者画像

| 角色 | 动机 | 常用手段 |
|---|---|---|
| **闪贷套利者** | 一笔交易内利用价差 | 闪贷 + 价格预言机操纵 |
| **MEV 搜寻者** | 抢跑 / 三明治交易 | 监听 mempool + 高 gas 价 |
| **审计绕过型** | 找出代码缺陷 | 模糊测试 + 状态遍历 |
| **社工/钓鱼** | 骗 owner 签错误交易 | tx.origin 钓鱼合约 |
| **协议级攻击** | 跨合约权限提升 | delegatecall / 升级代理 |

## 漏洞利用的共通模式

绝大多数链上攻击可以套用这套思路：

1. **找到状态机的不变量**（如"用户余额之和应等于 totalSupply"）
2. **找到能破坏不变量的路径**（如重入造成多次提现）
3. **构造调用序列**让攻击者在不变量被破坏时占据有利位置
4. **撤离资金**到自己的地址

## 经济激励才是根本

智能合约安全的特别之处：**任何漏洞都自动转化为可直接变现的资金**。一旦合约持有了价值，全球任何懂技术的人都被激励去攻击它——没有 24 小时的响应窗口，没有"刚发现漏洞还来得及修"。

这意味着：

- **上线前必须审计**
- **永远不要在生产链上"先发上去再说"**
- **关键修复要走升级代理 + 时间锁**

## 下一节

「安全编码原则」会把这套威胁意识落到具体的编码习惯上：Checks-Effects-Interactions、最小权限、不变量断言。
